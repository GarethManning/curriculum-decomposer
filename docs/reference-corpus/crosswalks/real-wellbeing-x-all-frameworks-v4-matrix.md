**Reading the matrix**: A "—" in any column is a coverage gap for that framework at that band. `gap_count` = number of frameworks with "—" for that row. Alignment is bidirectional: gaps on both sides are flagged equally — no framework is the reference.

**Content notes**: REAL cells use first-person "I can…" summaries derived from unified-wellbeing-data-v6. For the 14 LTs carried over from v3 (LT 1.1, 1.2, 2.1, 2.2, 3.1, 3.2, 4.1, 4.2, 5.1, 5.2, 6.1, 6.2, 7.1, 7.2), REAL cells are preserved verbatim from v3 where the LT content did not change. External framework cells are preserved verbatim from v3 (pipeline-generated from the actual source texts in docs/reference-corpus/{uk-statutory-rshe,welsh-cfw-health-wellbeing,casel-sel-continuum,circle-solutions-sel}/source.md). `†` indicates a representative LT label rather than a verbatim source quote.

**Band source abbreviations**: EoP = End of Primary; EoS = End of Secondary; PS = Progression Step; PK = Pre-K; K-1 = K–Grade 1; G2-3 = Grade Band 2–3; G4-5 = Grade Band 4–5; G6-8 = Grade Band 6–8; G9-10 = Grade Band 9–10; G11-12 = Grade Band 11–12; Y2 = Year 2; Y6 = Year 6; Y9 = Year 9; Y12 = Year 12.

**Alignment-statement convention added (new in v4).** Each LT × external-framework pair is now labelled with one of four alignment forms — `aligned-with-reciprocal-treatment`, `partial-alignment`, `reversed`, `absent` — in a new section "Per-LT alignment forms" at the bottom of this document. The 23-theme matrix retains v3's prose-cell convention for continuity; the new forms supplement, not replace, that convention.

**Bidirectional gap flagging explicit.** v3 flagged REAL gaps (T15, T22, T16, T17, T18, T19, T20) and noted REAL distinctive strengths (T12, T13, T14). v4 flags gaps in both directions symmetrically — external gaps in any framework (not only REAL) are now visible at theme level.

**REAL coverage gaps closed by new LTs:**
- **T15 (Personal Identity & Self-Concept)** — v3 marked REAL "—" across all six bands and flagged `T15_real_gap_confirmed`. LT 1.3 now supplies REAL content A–F. Gap closed.
- **T18 (Bullying, Stereotyping & Anti-Discrimination)** — v3 marked REAL "—" across all bands. LT 4.3 now supplies REAL content A–F. Gap closed.
- **T17 (Online Safety & Digital Wellbeing)** — v3 showed REAL partial coverage at C (LT 4.1) and E (LT 2.1) only. LT 8.1 + LT 8.2 + LT 8.3 now supply systematic REAL coverage across A–F. Gap substantially closed; REAL's online-safety content is now distinctive rather than absent.

**REAL coverage gaps still open (deliberate or pending):**
- **T22 (Growth Mindset & Self-Efficacy)** — REAL still has no dedicated LT for this construct. LT 7.2 covers adjacent dispositional territory but does not name or teach growth mindset as a construct. Flag preserved.
- **T16 (Family Diversity & Relationship Structures)** — REAL still has no LT. Statutory RSHE content; out-of-scope for REAL by deliberate design, per Competency 4 authoring note.
- **T19 (Physical Activity & Movement)** — REAL still has no LT (PE delivery context).
- **T20 (Nutrition & Food Literacy)** — REAL still has no standalone LT (LT 3.1 integrates briefly).

**REAL draft markers dropped.** v3 marked Bands E/F REAL content as *(draft)* pending teacher review. REAL-5 QA Step 6 has since completed with framework mean 90.3 PASS. v4 drops the blanket *(draft)* marker. One residual flag preserved: LT 4.2 Band D/F (deferred from Step 6 pending a unit-plan session — no content change until then). LT 4.4 Band D descriptor refinements (REAL-6b) and LT 4.5 authoring (REAL-8b/c/d) completed post-v3.

**Source provenance.** v3 drew REAL content from `band-tagged-real-wellbeing-v1.json` (REAL-2a). v4 draws REAL content from `unified-wellbeing-data-v6.json` (REAL-8d). 14-LT content is substantively the same between the two; the change is additive (new LTs).

**Convention inherited without change.** Theme codes T01–T23 and their labels (same 23 themes as v3); matrix column ordering (Theme | Band | REAL | RSHE | Welsh CfW | CASEL SEL | Circle Solutions | gap_count); verbatim-quote policy on external framework cells (`†` for LT-label substitutes); skill_flags and theme_grouping_flags sections at document bottom; no overwriting of v3 matrix files.
